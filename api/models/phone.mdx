## Overview

The Phone model represents a phone number associated with a user account. It includes the phone number along with city and country codes for international support.

## Model Structure

### Core Model

The core domain model is defined in `Phone.java`:

```java Phone.java
package com.fbaron.core.model.phone;

import lombok.Builder;
import lombok.Data;

@Data
@Builder
public class Phone {
    private long id;
    private long number;
    private int cityCode;
    private String countryCode;
}
```

## Fields

<ResponseField name="id" type="long">
  Unique identifier for the phone record. Auto-generated by the database.
</ResponseField>

<ResponseField name="number" type="long" required>
  The phone number without country or city code.
  
  <Expandable title="Example">
    For the phone number +1 (212) 555-1234:
    - `number`: 5551234
    - `cityCode`: 212
    - `countryCode`: "+1"
  </Expandable>
</ResponseField>

<ResponseField name="cityCode" type="int" required>
  The area or city code for the phone number.
  
  <Note>
    Store as an integer without leading zeros. Example: 212, not 0212.
  </Note>
</ResponseField>

<ResponseField name="countryCode" type="String" required>
  The international country calling code, including the '+' prefix.
  
  <Expandable title="Common Country Codes">
    - United States/Canada: `"+1"`
    - United Kingdom: `"+44"`
    - Germany: `"+49"`
    - France: `"+33"`
    - Japan: `"+81"`
  </Expandable>
</ResponseField>

## API Representation (DTO)

When interacting with the REST API, phone numbers are represented using the `PhoneDTO` class:

```java PhoneDTO.java
package com.fbaron.web.dto.phone;

public class PhoneDTO {
    private long number;
    private int cityCode;
    private String countryCode;
}
```

<Note>
  Notice that the `id` field is not included in the DTO. The ID is managed internally and not exposed in API requests/responses for phone creation.
</Note>

## Database Entity

The Phone model is persisted using JPA with the `PhoneEntity` class:

<Expandable title="PhoneEntity.java">
```java
package com.fbaron.data.jpa.phone.entity;

import javax.persistence.*;

@Entity
@Table(name = "Phone")
public class PhoneEntity {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private long id;
    
    private long number;
    private int cityCode;
    private String countryCode;
}
```
</Expandable>

## Relationship with User

Phone numbers are associated with users through a one-to-many relationship:

- Each User can have multiple Phone numbers
- Phone numbers are cascade-persisted with the User entity
- When a User is created or updated, associated Phone numbers are automatically managed

<Note>
  Phone numbers cannot exist independently. They must be associated with a User account.
</Note>

## Validation Considerations

When implementing phone number validation, consider:

- **Number format**: Ensure the number contains only digits
- **Country code format**: Should always include the '+' prefix
- **City code validity**: Verify the city code is valid for the given country
- **Number length**: Different countries have different valid phone number lengths
