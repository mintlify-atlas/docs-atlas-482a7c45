## Overview

The Login endpoint allows you to authenticate an existing user by providing their authentication token. The token is obtained during the sign-up process.

<Note>
  This endpoint requires the user's token obtained from the Sign Up endpoint.
</Note>

## Endpoint

```
POST /user-management/api/v1/login
```

## Request Headers

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`
</ParamField>

<ParamField header="Accept" type="string" required>
  Must be `application/json`
</ParamField>

## Request Body

<ParamField body="token" type="string" required>
  The authentication token for the user. This token is generated during sign-up and must be provided to log in.
</ParamField>

## Response

<ResponseField name="id" type="UUID">
  Unique identifier of the logged-in user
</ResponseField>

<ResponseField name="created" type="string">
  Timestamp when the user was originally created (format: "MMM dd, yyyy hh:mm:ss a")
</ResponseField>

<ResponseField name="lastLogin" type="string">
  Updated timestamp of the user's current login (format: "MMM dd, yyyy hh:mm:ss a")
</ResponseField>

<ResponseField name="token" type="string">
  The user's authentication token
</ResponseField>

<ResponseField name="isActive" type="boolean">
  Indicates whether the user account is active
</ResponseField>

<ResponseField name="name" type="string">
  The user's full name
</ResponseField>

<ResponseField name="email" type="string">
  The user's email address
</ResponseField>

<ResponseField name="password" type="string">
  The user's hashed password
</ResponseField>

<ResponseField name="phones" type="array">
  List of phone numbers associated with the user
  <Expandable title="Phone object structure">
    <ResponseField name="number" type="long">
      The phone number
    </ResponseField>
    <ResponseField name="cityCode" type="integer">
      The city or area code
    </ResponseField>
    <ResponseField name="countryCode" type="string">
      The country code
    </ResponseField>
  </Expandable>
</ResponseField>

## Implementation Reference

The Login endpoint is implemented in:
- Controller: `com.fbaron.web.rest.LoginRestAdapter` (LoginRestAdapter.java:67)
- Request DTO: `com.fbaron.web.dto.user.LoginDTO` (LoginDTO.java)
- Response DTO: `com.fbaron.web.dto.user.UserDTO` (UserDTO.java)

## Authentication Flow

1. **Sign Up**: User registers via the `/sign-up` endpoint and receives a token
2. **Save Token**: Client application stores the token securely
3. **Login**: User authenticates by providing the token to this endpoint
4. **Access Granted**: Server validates the token and returns updated user information with `lastLogin` timestamp

<Tip>
  The `lastLogin` field is automatically updated each time a user successfully logs in.
</Tip>

<Warning>
  Keep the authentication token secure and never expose it in client-side code or logs. Treat it like a password.
</Warning>
