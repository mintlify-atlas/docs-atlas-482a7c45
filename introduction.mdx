## Overview

The User Management Microservice is a demo Spring Boot application that provides secure user registration and authentication capabilities. Built with hexagonal architecture principles, it offers a clean separation of concerns between business logic and external dependencies.

## Technology Stack

The microservice is built with modern Java technologies:

### Core Framework
- **Spring Boot** - Application framework with embedded server
- **Spring Security** - Authentication and authorization
- **Spring Data JPA** - Data persistence layer
- **Java 11** - Programming language

### Key Dependencies


## Architecture

This application follows **Hexagonal Architecture** (also known as Ports and Adapters), which decouples the core business logic from external dependencies like databases, UI frameworks, and other systems.

<Note>
The application is divided into three main layers:
- **Web Layer** (External): Handles HTTP communication and REST endpoints
- **Core Layer** (Internal): Contains business logic, domain models, and use cases
- **Data Source Layer** (External): Manages data persistence and external integrations
</Note>

### Layer Breakdown

**Core Layer**
- `Model`: Domain entities representing business concepts
- `Domain`: Domain services with complex business logic
- `Service`: Use case implementations for specific user actions
- `Repo`: Repository interfaces for data access

**Web Layer**
- REST controllers (adapters) for HTTP endpoints
- DTOs for request/response serialization
- Exception handling advice

**Data Source Layer**
- JPA repositories and entities
- Database configuration
- Security and JWT configuration

## Server Configuration

The application runs with the following default configuration:

```yaml application.yaml
server:
  port: 8080
  servlet:
    context-path: /user-management

jwt:
  issuer: fbaron.com
  secret: eac4fdc0548e4a8c9ef9ae599b498ec2
  expiration: 86400000  # 24 hours in milliseconds
```

<Warning>
The JWT secret shown above is for demonstration purposes only. In production, use a secure secret stored in environment variables.
</Warning>
